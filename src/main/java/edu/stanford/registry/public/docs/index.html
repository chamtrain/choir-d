CHOIR API Documentation

The CHOIR API is designed to provide both access to CHOIR data as well as integration points within the CHOIR clinic application. The data exchange portion of the API and the integration interfaces can be used separately or in concert.

Data exchange

The data exchange portion of the CHOIR API is comprised of RESTful services that read and/or return JSON objects.

Version 1.0 of the API will support GET requests for “Patient”, “PatientAttribute”, “PatientReport”, “Survey” and “Assessment” resources.

See the Restlet API documentation for details

The API uses the same security mechanisms as the application. Version 1.0 of the API requires the “Schedule, Patient & Report tabs” (ROLE_CLINIC_STAFF) permission.

CHOIR interface integration points:

Integration points within the application provide a way to include custom data generated by HTML within the CHOIR interface.

Using JavaScript or a JavaScript framework like AngularJS or React custom html can retrieve and display CHOIR data through the data API and optionally combine it with external static and dynamic content.

See the Interface API documentation for details.
